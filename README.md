# Ball Mixer Game

Небольшая 2D-игра про шарики на Python и pygame-ce.

- На поле летают цветные шарики.
- При столкновении шарики **смешивают цвета**, но не отталкиваются физически.
- ЛКМ можно «всасывать» шарик в инвентарь и переносить его мышью.
- ПКМ можно отправлять шарики в инвентарь.
- Колёсиком мыши можно «выплёвывать» шарики обратно на поле из инвентаря.
- В левом верхнем углу есть зона удаления: шарики, залетающие туда, пропадают.

Игровая логика вынесена в файл `logic.py`, визуальный интерфейс — в `gui.py`.

---

## 1. Локальный запуск (без Docker)

### 1.1. Требования

- Python 3.10+ (подойдёт и более свежая версия).
- Установленный `pygame-ce` (fork pygame) [web:53].
- Рабочая графическая среда (Windows / macOS / Linux с X11).

Создай и активируй виртуальное окружение (опционально):

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate
```

Установи зависимости:

```bash
pip install pygame-ce
```

Если хочешь фиксировать зависимости, создай `requirements.txt`:

```text
pygame-ce==2.5.7
```

и установи:

```bash
pip install -r requirements.txt
```

### 1.2. Запуск игры

Убедись, что рядом лежат файлы:

- `logic.py` — модуль с игровой логикой.
- `gui.py` — точка входа с графикой.

Запусти:

```bash
python gui.py
```

Откроется окно с белым фоном и анимированными шариками.

## 2. Docker: сборка и запуск с графикой (X11, Linux)

Ниже пример для Linux с X11 (Ubuntu / Debian и т.п.) с пробросом X-сокета [web:63][web:66][web:69][web:70].

### 2.1. Требования

- Установленный Docker.
- Запущенный X11-сервер (обычно уже есть на десктопных Linux-системах).
- Пользователь в группе `docker` (чтобы не писать `sudo` на каждый запуск — опционально).

### 2.2. Dockerfile

В корне проекта лежит `Dockerfile` (см. ниже в этом репо).  
Он делает следующее:

- Базовый образ: `python:3.12-slim`.
- Устанавливает системные библиотеки, нужные pygame-ce (SDL2, шрифты, X11 и звук) [web:64][web:70].
- Устанавливает Python-зависимости (`pygame-ce`).
- Копирует `logic.py`, `gui.py` и остальные файлы в образ.
- В качестве точки входа запускает `python gui.py`.

Сборка образа:

```bash
docker build -t ball-mixer-game .
```

### 2.3. Разрешить доступ к X-серверу

На хостовой машине (Linux) нужно разрешить контейнеру подключаться к X-серверу [web:63][web:66][web:69]:

```bash
xhost +local:docker
```

> Важно: после тестов можно закрыть доступ:
> ```bash
> xhost -local:docker
> ```

### 2.4. Запуск контейнера с GUI

Запусти контейнер с пробросом дисплея и X-сокета:

```bash
docker run --rm \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  --name ball-mixer-game \
  ball-mixer-game
```

- `-e DISPLAY=$DISPLAY` — передаёт внутрь контейнера адрес текущего дисплея [web:63][web:68].
- `-v /tmp/.X11-unix:/tmp/.X11-unix` — пробрасывает Unix-сокет X11, через который идёт вывод окна [web:63][web:69].
- `--rm` — удаляет контейнер после выхода из игры.

Если всё настроено, окно игры откроется на твоём рабочем столе, хотя код выполняется внутри Docker-контейнера.

## 3. Docker + Windows / macOS (заметка)

Для Windows и macOS схема сложнее: нужен X-сервер (например, XQuartz на macOS, VcXsrv или Xming на Windows), настройка `DISPLAY`, иногда socat-туннель и т.п. [web:61][web:64][web:70].  
В этом README описан базовый вариант для Linux/X11. Для других систем логику можно адаптировать по этим же принципам.

## 4. Управление в игре

- ЛКМ:
  - Клик по шарику — взять его (он «прилипает» к курсору).
  - Отпустить ЛКМ — выпустить шарик обратно с импульсом скорости.
- ПКМ:
  - Клик по шарику — отправить его в инвентарь.
  - Клик по пустому месту, когда шарик в руке — тоже отправить его в инвентарь.
- Колёсико мыши:
  - Прокрутка — выплюнуть один шарик из инвентаря в точке курсора.
- Зона удаления:
  - Шарики, которые залетают в выделенный прямоугольник в левом верхнем углу, удаляются.

## 5. Структура проекта

- `logic.py` — игровая логика: движение, столкновения, смешивание цветов, инвентарь, удаление.
- `gui.py` — графический интерфейс, работа с pygame-ce, отрисовка и обработка мыши.
- `Dockerfile` — образ для запуска игры в контейнере с X11.
- `README.md` — этот файл.
